# 微信排版规范

## ⚠️ 关键约束

微信公众号草稿箱会剥离 `<div>` 容器样式。之前用的 :::block 模块（hero/verdict/callout/cta）在本地预览时正常，但推送到微信草稿后，框的背景、边框、圆角会消失，仅剩裸文字，破坏阅读体验。**绝对不要再使用 :::block 容器类模块。**

## 核心原则

**内容第一，样式第二。** 排版是为了提升阅读体验。微信原生支持 ## 标题、**加粗**、> 引用、--- 分割线、- 列表——这些最稳定，优先使用。

---

## 标题防重复规则（重要）

- **正文中不要写 # 文章主标题（H1）**：微信公众号草稿栏会自动显示文章标题
- **正文中不要重复出现文章标题**：文章标题已在微信标题栏展示，正文中任何位置都不应再次出现与文章标题相同的文字
- **正文中只用 ##（H2）和 ###（H3）做分节标题**
- **正文首个 ## H2 如果与 --title 相同，converter 会自动去掉**

---

## WeChat 安全排版方案

| 需求 | 之前的不安全写法 (:::) | 替换为安全的原生 Markdown |
|------|----------------------|--------------------------|
| **文章开头** | :::hero 带 title/eyebrow/subtitle | > + 开场引语，或直接正文 |
| **强调核心观点** | :::verdict 卡片 | **💡 核心：** + 加粗文字 |
| **提示/引用** | :::callout 卡片 | > 💡 + 提示内容（微信原生 blockquote） |
| **行动号召** | :::cta 卡片 | --- 分隔线 + **📌** + 结语文字 |
| **问答列表** | :::faq 卡片 | **Q：**xxx + 换行 + **A：**xxx |

---

## 排版语法速查

```
> 开场引语或副标题

**💡 核心：** 加粗突出文章的中心判断。

> 💡 提示或补充说明（替代 callout）

---

**📌 结语：** 行动号召文字（替代 cta）
```

## 重点内容强调样式（重要）

| 强调级别 | 写法 | 适用场景 |
|---------|------|---------|
| **核心观点** | `**💡 核心：** 观点内容` | 每篇文章1-2次，突出最关键判断 |
| **重点结论** | `**结论：**` 或 `**关键发现：**` | 章节末尾总结 |
| **重要数据/事实** | `**数字加粗**` | 关键数字、时间、版本号 |
| **警告/坑点** | `⚠️ **注意：**` 或 `**坑点：**` | 踩坑经验、易错提醒 |
| **对比强调** | `✅ 正确做法` / `❌ 错误做法` | 方案对比 |
| **经验总结** | `📌 **经验：**` | 实战经验提炼 |

**使用原则：**
- **每个章节至少1处加粗强调**：避免整章纯文字无视觉锚点
- **加粗占比 5-15%**：太少看不出重点，太多等于没加粗
- **emoji + 加粗组合**：💡📌⚠️✅❌ 搭配加粗，视觉区分度高
- **不要只加粗单个词**：至少加粗一个完整短语或句子

---

## 排版使用原则

- **⚠️ 绝对不要使用 :::block 容器模块**（hero/verdict/callout/cta/toc/faq）
- **⚠️ 正文不要写 # 文章主标题（H1）**
- **⚠️ 正文不要重复出现文章标题**
- **优先原生语法**：## 标题、**加粗**、> 引用、--- 分割线、- 列表
- **emoji + 加粗辅助视觉**：💡📌📖⚠️✅❌ 等 emoji 搭配加粗
- **20000 字符上限**：原生语法字符占用少，更多空间留给正文

## 示例：纯 Markdown -> 增强后

**用户写的原始 Markdown：**
```markdown
## 为什么你的文章读者读不完

你有没有花了一整天写的好文章，发出去后阅读量惨淡？

问题可能不是内容，而是排版。好的排版要在 3 秒内让读者决定继续读。

## 排版的核心

排版的本质不是审美，而是降低读者的决策成本。每一次在手机上阅读，读者都在做决策：继续还是划走。

## 开始行动

从今天开始重构你的公众号文章吧。
```

**💡 核心：** 排版的本质是降低读者的决策成本。每一次在手机上阅读，读者都在做决策：继续还是划走。排版要做的是让"继续"这个决策更容易。

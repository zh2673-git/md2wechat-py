# 公众号发布工具 - 架构决策

## 一、技术选型

| 决策项 | 选择 | 推导逻辑 |
|-------|------|---------|
| **快速原型语言** | Python 3.11+ | 开发效率高、AI IDE 生态好、微信/prompt/图片 SDK 丰富、快速验证业务逻辑 |
| **生产部署语言** | Go 1.22+ | 单二进制分发、跨平台、启动快、可复用 md2wechat 架构设计 |
| **MD→HTML 转换** | markdown-it (Python) / goldmark (Go) | 标准 Markdown 解析器，扩展性好 |
| **`:::block` 排版解析** | 自研 PEG parser | 专有语法，独立实现，不依赖第三方 API |
| **主题系统** | CSS 变量 + YAML 定义 | 轻量、无需构建工具、用户可自定义 |
| **微信 SDK** | 自封装（参考 wechatpy） | 只用了 token + 素材 + 草稿 3 个 API，不需要全量 SDK |
| **AI 图片** | httpx requests + 统一 provider 接口 | 类似 md2wechat 的 provider 抽象 |
| **CLI 框架** | Python: click / Go: cobra | 各自生态标准 |
| **MCP 协议** | Python mcp 库 | 跨 IDE 统一工具协议 |
| **配置管理** | YAML 文件 + 环境变量 | 与 md2wechat 一致，用户习惯 |

## 二、为什么不是直接 Fork md2wechat？

| 维度 | Fork md2wechat | 自研 |
|------|---------------|------|
| **排版引擎** | 依赖作者的 API 服务（付费） | 完全自研，自部署 |
| **扩展性** | 受限于上游架构 | 按需设计，灵活调整 |
| **跨平台** | 只有 Claude Code/OpenClaw | 主动设计 CodeBuddy/Trae/Cursor/MCP |
| **技术栈** | 仅 Go | Python 原型 + Go 生产 双轨 |
| **学习成本** | 理解 Go 源码 | 自己设计、完全掌控 |

## 三、设计原则

1. **核心引擎独立于 IDE** — 排版引擎、微信适配、图片引擎都是独立包，不绑定任何 AI IDE
2. **薄适配层** — 每个平台只有几十行的适配代码（SKILL.md / CLAUDE.md / rules），核心逻辑在引擎中
3. **确定性优先** — `:::block` 排版结果一致，不像 prompt 每次不同
4. **可发现** — 所有能力和资源可通过 CLI 发现，Agent 不需要猜
5. **确认层先行** — 发布前 inspect → validate → confirm，降低错误率

## 四、与其他方案对比

| 方案 | 自研 | Fork md2wechat | 纯用微信编辑器 |
|------|------|---------------|--------------|
| **自主可控** | ✅ 完全 | ❌ 依赖上游API | ✅ |
| **开发工作量** | 中高 | 低（直接编译） | 零 |
| **AI IDE 集成** | ✅ 全面设计 | ✅ 已有 | ❌ |
| **排版能力** | ✅ 43组件自研 | ✅ 43组件付费 | ❌ 手动排版 |
| **扩展性** | ✅ 自由 | ⚠️ 受上游限制 | ❌ |

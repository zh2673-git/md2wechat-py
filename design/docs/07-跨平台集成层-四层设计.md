# 跨平台集成层 - 四层设计

## 模块内部状态

```python
@dataclass
class IntegrationState:
    """集成层内部状态"""
    current_platform: str = ""     # codebuddy | claude | trae | mcp
    cli_binary_path: str = ""      # 统一 CLI 路径
    skill_dir: str = ""            # 技能文件目录
```

## 四层设计

| 层面 | 设计内容 |
|-----|---------|
| **数据规矩** | 每个平台的适配规范：CodeBuddy→`SKILL.md` + `workflows/`；Claude Code→`CLAUDE.md` + `skills/`；Trae→`.trae/rules/` + MCP；统一：`CLI --json` 输出格式一致 |
| **数据存储** | 适配文件存各自平台约定位置；核心引擎统一通过 `pip install wechat-pub` 或 `go install` 安装 |
| **数据流转** | `用户自然语言 → AI IDE识别意图 → 匹配skill描述 → 构建CLI命令 → 执行 → 解析JSON envelope → 呈现给用户` |
| **接口层** | `wechat-pub` CLI 是所有平台的**唯一后端接口**，平台适配层只负责将自然语言意图映射为 CLI 命令 |

## 支持的平台适配

### 1. CodeBuddy

```markdown
.codebuddy/skills/公众号skill/SKILL.md
.codebuddy/skills/公众号skill/workflows/完整发布.md
.codebuddy/skills/公众号skill/workflows/快速排版.md
.codebuddy/skills/公众号skill/workflows/图片处理.md
```

CodeBuddy 技能格式：YAML frontmatter 定义 skill 元数据 → Markdown 正文定义命令和工作流。

### 2. Claude Code

```markdown
CLAUDE.md
```

或通过 `npx skills add` 安装标准的 skill 包。

### 3. Trae / Cursor

```yaml
.trae/rules/wechat-pub.md
```

Trae 支持自定义 rules 文件（Markdown），定义了 AI 如何理解项目和处理用户请求。

### 4. MCP Server（通用协议）

MCP Server 是基于 **stdio 的通用协议**，任何支持 MCP 的 IDE（包括 Trae、Cursor、VS Code 的 GitHub Copilot 等）都能连接：

```json
{
  "mcpServers": {
    "wechat-pub": {
      "command": "wechat-pub-mcp",
      "args": [],
      "env": {}
    }
  }
}
```

# 公众号发布工具

通过 `wechat-pub` CLI 完成 Markdown → 微信公众号的全流程发布。

## 安装
```bash
pip install wechat-pub
wechat-pub config init
```

## 命令

| 目的 | 命令 |
|------|------|
| 检查文章 | `wechat-pub inspect article.md` |
| 预览排版 | `wechat-pub preview article.md` |
| 转换HTML | `wechat-pub convert article.md -o out.html` |
| 推送草稿 | `wechat-pub convert article.md --draft --cover cover.jpg` |
| 排版模块 | `wechat-pub layout list/show/render/validate` |
| 生成封面 | `wechat-pub generate_cover --article article.md` |
| AI 去痕 | `wechat-pub humanize article.md` |

## 工作流

### 完整发布
```bash
wechat-pub inspect article.md           # 1. 检查
wechat-pub preview article.md           # 2. 预览
wechat-pub convert article.md --draft --cover cover.jpg  # 3. 推送
```

### 排版模块
```bash
wechat-pub layout list --serves attention
wechat-pub layout show hero
wechat-pub layout render hero --var title="标题" --var eyebrow="观察"
wechat-pub layout validate --file article.md
```

## Agent 规则
1. 先执行 `wechat-pub capabilities --json` 发现能力
2. inspect → preview → convert 的安全流程
3. 排版模块每类最多1个
```

### Trae Rules 文件

```markdown
# .trae/rules/wechat-pub.md

## 公众号发布工具使用指南

本项目提供 `wechat-pub` CLI 工具，将 Markdown 转换为微信公众号排版并推送草稿箱。

### 可用命令
- `wechat-pub inspect <file>` — 检查文章发布准备状态
- `wechat-pub preview <file>` — 本地预览排版效果
- `wechat-pub convert <file> [--draft] [--cover path]` — 转换并选择推送
- `wechat-pub layout list/show/render/validate` — 排版模块
- `wechat-pub generate_cover --article <file>` — AI 生成封面
- `wechat-pub generate_infographic --article <file>` — AI 生成信息图

### 工作流
用户说"发布文章"时：执行 inspect → preview → convert --draft 三步
用户说"排版"时：检查文章中是否有 :::block 语法 → layout validate → convert
用户说"配图"时：generate_cover → generate_infographic
```

### MCP Server 定义

```python
# mcp_server.py - 基于 MCP 协议的标准工具服务
from mcp.server import Server
from mcp.types import Tool, TextContent

server = Server("wechat-pub")

@server.list_tools()
async def list_tools():
    return [
        Tool(
            name="convert",
            description="Markdown → 微信公众号 HTML",
            inputSchema={
                "type": "object",
                "properties": {
                    "markdown": {"type": "string"},
                    "theme": {"type": "string", "default": "default"},
                    "output": {"type": "string"}
                }
            }
        ),
        Tool(
            name="publish_draft",
            description="推送草稿到微信",
            inputSchema={
                "type": "object",
                "properties": {
                    "md_file": {"type": "string"},
                    "cover": {"type": "string"}
                }
            }
        ),
        Tool(
            name="generate_cover",
            description="AI 生成封面图",
            inputSchema={
                "type": "object",
                "properties": {
                    "article_file": {"type": "string"}
                }
            }
        ),
        Tool(
            name="layout_list",
            description="列出排版模块",
            inputSchema={
                "type": "object",
                "properties": {
                    "serves": {"type": "string", "enum": ["attention", "readability", "memorability", "conversion"]}
                }
            }
        ),
    ]
```

# 公众号发布工具 - 本质分析

## 一、是什么（坡度三阶段）

### 阶段1：点出概念

> **一句话定义**：公众号发布工具是一个「Markdown 公众号内容工厂」— 写 Markdown，自动排版成微信公众号图文，AI 配图，一键推送草稿箱。面向所有 AI IDE 和开发者。

**本质特征**：

| 推导要素 | 逻辑分析 | 结论 |
|---------|---------|------|
| **核心问题** | 公众号排版费时、流程碎片化、AI 工具链碎片化 | 一个命令完成"写→排→图→发" |
| **目标用户** | 公众号创作者 + AI IDE 用户（CodeBuddy/Trae/Claude等） | 无需离开编辑器 |
| **核心价值** | 从 Markdown 到微信草稿箱，一键直达 | 发布效率提升 10 倍 |
| **输出形式** | CLI + 平台技能适配 + MCP 协议 | 所有 AI IDE 可用 |

### 阶段2：建立联系

| 属性名 | 推导来源 | 类型 | 说明 | 深入？ |
|-------|---------|-----|------|-------|
| **Markdown 转换引擎** | 核心转换 | 模块 | MD → 微信兼容 HTML | ✅ 需深入 |
| **排版组件库** | 美观排版 | 模块 | 43 个 `:::block` 视觉组件 | ✅ 需深入 |
| **主题系统** | 统一视觉 | 模块 | 40+ 微信精调 CSS 主题 | ❌ 当前说明 |
| **微信适配层** | 推送草稿 | 模块 | 微信 API + 素材上传 + 草稿 | ❌ 当前说明 |
| **AI 配图引擎** | 自动配图 | 模块 | 封面/信息图/文章配图 | ❌ 当前说明 |
| **跨平台技能层** | 多 IDE 适配 | 模块 | CodeBuddy/Claude/Trae/MCP | ✅ 需深入 |
| **确认层** | 安全发布 | 模块 | inspect→preview→convert | ❌ 当前说明 |

### 阶段3：详细解释

**区分属性（边界）**：
- ✅ **做**：MD→HTML、排版模块、主题、AI 配图、微信草稿推送、跨 IDE 集成
- ❌ **不做**：内容管理 CMS、数据分析、多用户平台、微信已发布文章管理

**设计哲学**：**复用成熟引擎 + 自建自有能力 + 跨平台适配**
1. 排版模块系统：从 md2wechat 学习设计但**完全自研**
2. 微信适配：直接封装微信官方 API（开源可用的 SDK）
3. 跨平台：为每个 AI IDE 写薄适配层，共享同一个 Python/Go 内核

## 二、为什么（坡度三阶段）

### 阶段1：点出概念

> **存在理由一句话总结**：现有 md2wechat 的 API 模式是付费闭源的，你需要的是一套完全自有的、开放的全流程公众号发布工具链，不依赖任何第三方 API Key。

### 阶段2：建立联系

| 决策项 | 选择 | 推导逻辑 |
|-------|------|---------|
| **核心语言（快速原型）** | Python | 开发效率高、AI IDE 生态好、微信 SDK 和图片 SDK 丰富 |
| **核心语言（生产部署）** | Go | 单二进制、启动快、跨平台，参考 md2wechat 的设计 |
| **排版模块** | 自研 `:::block` 解析 → HTML 渲染引擎 | 避免依赖第三方付费 API |
| **主题系统** | CSS 变量 + 预置 YAML 主题 | 轻量、可扩展、用户可自定义 |
| **微信 SDK** | 封装 `wechatpy` (Python) / 自写 (Go) | 微信官方 API 文档即可实现 |
| **AI 图片** | 对接 Volcengine/OpenAI 等 API | 与 md2wechat 一样，灵活选择 |
| **确认层** | inspect→render→validate 三步 | 从 md2wechat 学到的安全设计 |
| **跨平台** | 薄适配层：CodeBuddy SKILL.md + Trae rules + MCP Server | 一份核心引擎，多平台共用 |

## 三、怎么做（坡度三阶段）

### 阶段1：点出概念

> **系统架构一句话概述**：分层架构 — 核心引擎（converter + layout + image + wechat）被统一 CLI 包装，再为每个 AI IDE 编写薄适配层。

### 阶段2：建立联系

**模块划分与接口契约**：

```
项目（公众号发布工具）
│
├── 排版引擎 ✅ 需深入
│   ├── 内部状态: modules(YAML注册), theme(CSS变量)
│   └── 对外接口: convert(), inspect(), preview(), layout_list/show/render/validate()
│
├── 微信适配层
│   ├── 内部状态: access_token(缓存), app_id, app_secret
│   └── 对外接口: upload_image(), create_draft(), upload_cover()
│
├── AI 图片引擎
│   ├── 内部状态: provider_config, model, preset
│   └── 对外接口: generate_cover(), generate_infographic(), generate_image()
│
├── 跨平台集成层 ✅ 需深入
│   ├── CodeBuddy: SKILL.md + workflows
│   ├── Claude Code: CLAUDE.md + skills/
│   ├── Trae: .trae/rules/ + MCP
│   └── MCP Server: stdio模式工具服务
│
└── 确认层
    ├── inspect: metadata + readiness + checks
    └── preview: 本地HTML预览
```

### 阶段3：详细讲解

**3.1 全局状态设计**

```python
# 统一的发布流程全局状态
@dataclass
class PublishState:
    # 当前会话
    work_dir: str
    md_file: str                  # 正在处理
    wechat_configured: bool
    image_configured: bool
    
    # 流程阶段
    stage: str = "idle"           # idle→inspected→previewed→converted→drafted
    
    # 产出
    html_output: str = ""
    cover_media_id: str = ""
    draft_media_id: str = ""
    
    # 错误
    errors: list = field(default_factory=list)
    warnings: list = field(default_factory=list)
```

**3.2 方法设计（状态流转）**

```
idle → inspect → preview → convert → draft → [done]
  │       │          │          │         │
  │       检查元数据  生成预览   转换MD→HTML 推送草稿箱
  │       readiness   HTML      上传图片
  │       checks               注入主题
```

**3.3 路由设计（多场景分发）**

| 场景 | 路由路径 | 说明 |
|------|---------|------|
| 快速转换 | `convert` | 仅转换 HTML |
| 完整发布 | `inspect → preview → convert --draft` | 带确认层安全发布 |
| 纯图片发布 | `image generate → upload → create_image_post` | 小绿书/图文笔记 |
| 风格写作 | `write → humanize → inspect → convert --draft` | AI 辅助创作全流程 |

**3.4 统一接口层**

CLI 输出统一 JSON envelope（借鉴 md2wechat 的契约）：

```json
{
  "success": true,
  "code": "OK",
  "message": "Success",
  "schema_version": "v1",
  "status": "completed",
  "retryable": false,
  "data": {}
}
```

状态码设计：
- `OK` - 成功
- `INSPECT_COMPLETED` - 检查完成
- `PREVIEW_READY` - 预览就绪
- `CONVERT_COMPLETED` - 转换完成
- `DRAFT_CREATED` - 草稿创建成功
- `CONFIG_INVALID` - 配置无效
- `WECHAT_TOKEN_FAILED` - 微信 token 获取失败
